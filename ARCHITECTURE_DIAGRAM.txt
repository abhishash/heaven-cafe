                       FILE DEPENDENCIES

Page (app/checkout/page.tsx)

├── Exports: stripe client
├── Used by: /app/actions/stripe.ts

/lib/razorpay.ts
├── Exports: razorpay utilities

/app/actions/stripe.ts
├── Creates checkout session
├── Validates order

/app/actions/razorpay.ts

└── Called by: /components/RazorpayCheckout.tsx


/context/CartContext.tsx
├── Manages cart state
├── Persists to localStorage
└── Consumed by: entire app


/app/order-confirmation/page.tsx

└── Has "Continue Shopping" link


================================================================================
                     PAYMENT PROCESSING FLOW
================================================================================

CASH ON DELIVERY FLOW:
─────────────────────

Customer Form Input
    ↓
Client-side Validation (email, phone, address)
    ↓
Submit Form
    ↓
handleCODSubmit() in /app/checkout/page.tsx
    ↓
Simulate processing (1 second delay)
    ↓
Log order to console (in production, save to DB)
    ↓
clearCart()
    ↓
Navigate to /order-confirmation?orderId=xxx&method=cod
    ↓
Order Confirmation Page
    ↓
Show delivery address
    ↓
Show "Payment at Delivery"


STRIPE PAYMENT FLOW:
────────────────────

Customer Selects Stripe
    ↓
<Checkout> Component renders
    ↓
Stripe Embedded Checkout appears
    ↓
Customer enters card details
    ↓
Clicks "Pay" button
    ↓
createStripeCheckoutSession() action
    ↓
createStripeCheckoutSession() validates amount
    ↓
Calls stripe.checkout.sessions.create()
    ↓
Returns clientSecret
    ↓
Stripe Modal processes payment
    ↓
Payment successful/failed callback
    ↓
handleStripeSuccess() or handleStripeError()
    ↓
clearCart()
    ↓
Navigate to /order-confirmation?orderId=xxx&method=stripe
    ↓
Order Confirmation Page


RAZORPAY PAYMENT FLOW:
──────────────────────

Customer Selects Razorpay
    ↓
<RazorpayCheckout> Component renders
    ↓
Click "Initialize Payment" button
    ↓
createRazorpayOrder() server action
    ↓
createRazorpayOrder() validates amount
    ↓
Creates order via Razorpay API
    ↓
Gets order ID from Razorpay
    ↓
Returns order details to client
    ↓
Razorpay Checkout Modal appears
    ↓
Customer selects payment method
    ↓
Customer completes payment
    ↓
Razorpay verifies payment signature
    ↓
handlePaymentSuccess() or handlePaymentError()
    ↓
clearCart()
    ↓
Navigate to /order-confirmation?orderId=xxx&method=razorpay
    ↓
Order Confirmation Page


================================================================================
                     API ENDPOINTS & ACTIONS
================================================================================

SERVER ACTIONS:
───────────────

/app/actions/stripe.ts
  └── createStripeCheckoutSession(amount, orderId)
      ├── Validates amount > 0
      ├── Creates Stripe session
      └── Returns clientSecret

/app/actions/razorpay.ts
  ├── createRazorpayOrder(amount, orderId)
  │   ├── Validates amount > 0
  │   ├── Creates Razorpay order
  │   └── Returns order details
  │
  └── verifyRazorpayPayment(paymentId, orderId)
      ├── Validates payment signature
      ├── Verifies payment status
      └── Returns verification result


PAYMENT PROVIDER CALLS:
───────────────────────

Stripe API:
  POST /v1/checkout/sessions
  └── Create secure payment session

Razorpay API:
  POST /api/v1/orders
  └── Create order for payment
  
  POST /api/v1/payments/{paymentId}/verify
  └── Verify payment signature

================================================================================
                           ERROR HANDLING
================================================================================

VALIDATION ERRORS:
  Form → Client-side validation → Show message to user

PAYMENT ERRORS:
  Payment Provider → Server-side verification → Show message to user

NETWORK ERRORS:
  Connection lost → Show error → User can retry

API ERRORS:
  Stripe/Razorpay API → Catch error → Show user-friendly message

DATABASE ERRORS (Future):
  DB connection → Log error → Show message to user
